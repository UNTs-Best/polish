# Contributing to Polish

## Branching
- Never push directly to `main`
- Create a new branch for every feature or fix
- Branch naming: `feature/your-feature-name` or `fix/your-fix-name`

## Workflow
1. Create a branch from `main`
2. Make your changes
3. Open a Pull Request
4. Wait for CI checks to pass
5. Request review from @namesarnav
6. Do not merge until approved

## Commit Messages
Keep them short and descriptive.
- `add resume upload endpoint`
- `fix auth token expiry bug`
- `update landing page copy`

Not:
- `fix`
- `changes`
- `asdfgh`

## Pull Requests
- Give your PR a clear title describing what it does
- If it fixes a bug or closes an issue, mention it in the description
- Keep PRs small and focused — one feature or fix per PR

## Code Style
- Run `npm run lint` before opening a PR
- Fix any errors before requesting review

## Questions
Reach out on the group chat before making large structural changes.
